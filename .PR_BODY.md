## 关键改动

### 1. LLM 多 Provider 抽象 (`src/models/`)
- `ModelClient` 接口统一：`isMock` / `chat` / `embed`
- 6 大 provider 工厂：OpenAI / Anthropic / DeepSeek / Zhipu / Moonshot / Ollama
- `detectProvider()` 启发式自动检测（API key 前缀 + 模型名）
- `getSupportedProviders()` 运行时枚举
- 集成测试 11 pass / 7 skip（`describe.skipIf(!hasAnyApiKey)` 模式，无 key 也能跑 mock）

### 2. E2E Skill 双路径生成 (`src/skills/builtin/e2e/index.ts`)
- **真实 LLM 路径** — `ModelClient.chat()` 调用 API
- **Template-based fallback** — 无 API key 时 keyword regex（中英双语）+ Playwright 模板
- `model.isMock` 自动检测
- `extractSelectors` regex 增强：支持 `getByRole('button', { name: 'X' })` 的 `name/label/text` 字段
- Unit 测试 29 pass

### 3. 性能 Skill 评分体系 (`src/skills/builtin/performance/index.ts`)

```ts
estimatePerformanceScore(diagnoses) → number  // 0-100
  - critical: -5, warning: -3, info: -1
  - >20 命中: 额外 -10
  - floor: 0

performanceGrade(score) → 'A'|'B'|'C'|'D'|'F'  // 90/80/70/50 分档

runLighthouseAudit(url) → throw LighthouseNotImplementedError
  // v3.1.1: 不再 silent null！调用方必须 try/catch 或用 estimatePerformanceScore
```

- Unit 测试 18 pass（含 throws 验证）

- **诊断**：二分法定位 → feedback-loop.test.ts + ab-testing.test.ts
- **修复**：移除 `mock.module` 调用，改用真实 `tmpDir` + `basePath` / `storageDir` 参数化
- **结果**：448 pass / 0 fail（修复前 437 / 11 fail）

### 5. Review 反馈修复（v3.1.1 patch）

#### 5.1 `evaluateDiagnosis` 边界 bug (`src/engines/harness/evaluation-engine/evaluators.ts`)
**3 个 bug 导致 4 个测试 fail**（PR body 之前误标"pre-existing"）：
- `falseNegatives` 用 `expectedTypes.size` 漏算 `issueCount`（a11y case 报 FN=1 应为 2）
- `precision` / `recall` 当 `TP=0` 时 vacuously 返回 1，应区分 `expectedTotal > 0` 边界
- FN 计算在 type-based 模式下不计入 matched actuals

**修复**：4 个测试从 fail → pass，整体 integration 0 fail

#### 5.2 Vue `unused-prop-vue` 检测失效（`src/utils/ast-analyzer.ts`）
**Vue F1 80% → 90% 根因**：
- `appearsInTemplateSection` 检查整段 source，因为 propName 总在 `<script>` 块的 `defineProps` 里 → 永远 true → 漏报
- `walkExpr` 没跳过 vue-eslint-parser 节点的 `parent` 引用 → 栈溢出

**修复**：
- 提取 `<template>...</template>` 段再正则
- 跳过 `parent/range/loc/start/end` 字段
- 深度上限 64

**效果**：Vue F1 80.0% → 90.0%（vue-007/008 F1: 0 → 1.0），整体 F1 89.8% → 90.5%

#### 5.3 `runLighthouseAudit` 静默 null（`src/skills/builtin/performance/index.ts`）
**问题**：v0.3.0 占位返回 `null` 让调用方 silent fallthrough，违反"显式优于隐式"
**修复**：抛 `LighthouseNotImplementedError`，错误信息指向 `estimatePerformanceScore` fallback
**效果**：调用方不能再假装 Lighthouse 跑过，避免用户被误导
